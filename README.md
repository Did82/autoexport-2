# AutoExport

AutoExport синхронизирует датированные каталоги из источника в хранилище,
контролирует заполнение обоих дисков и показывает журнал операций в веб-интерфейсе.

## Требования

- Bun 1.3.14 или совместимый;
- `rsync`;
- доступ на чтение к источнику и на запись к целевому каталогу.

## Локальный запуск

```bash
bun install
mkdir -p /tmp/src /tmp/dest
bun dev
```

Интерфейс и API доступны на <http://localhost:3001>.

Production-сборка:

```bash
bun run build
bun start
```

Полная проверка проекта:

```bash
bun run check
```

## Конфигурация

При первом запуске создаётся `config.json`:

```json
{
  "schemaVersion": 2,
  "src": "/tmp/src",
  "dest": "/tmp/dest",
  "srcLimit": 78,
  "destLimit": 78,
  "cleanupDays": 90,
  "quarantineDays": 7
}
```

`srcLimit` и `destLimit` — независимые проценты заполнения диска. Значение `100`
фактически отключает очистку по соответствующему порогу. Старый параметр `limit`
автоматически переносится в оба новых поля.

| Переменная | Значение по умолчанию | Назначение |
|---|---:|---|
| `SRC_PATH` | `/tmp/src` | Источник при первом запуске |
| `DEST_PATH` | `/tmp/dest` | Хранилище при первом запуске |
| `SRC_DISK_LIMIT` | `DISK_LIMIT` или `78` | Лимит источника |
| `DEST_DISK_LIMIT` | `DISK_LIMIT` или `78` | Лимит хранилища |
| `CLEANUP_DAYS` | `90` | Хранение журналов |
| `QUARANTINE_DAYS` | `7` | Хранение нестандартных папок |
| `APP_TIMEZONE` | `Europe/Minsk` | Часовой пояс всех cron-задач |
| `CONFIG_PATH` | `./config.json` | Путь к конфигурации |
| `DATABASE_PATH` | `./autoexport.db` | Путь к SQLite |
| `PORT` | `3001` | HTTP-порт |

Переменные окружения задают начальные значения. После создания `config.json`
настройки изменяются через интерфейс или API.

## Правила работы с каталогами

- Рабочее имя каталога — действительная дата в формате `YYYYMMDD`.
- `rsync` дополняет целевой каталог и не использует `--delete`.
- Каталог текущего дня никогда не удаляется автоматически.
- Перед удалением из источника выполняются финальная синхронизация и быстрый
  `rsync --dry-run` с проверкой размера и времени изменения.
- Нестандартные папки перемещаются в `.autoexport-quarantine` и удаляются после
  `quarantineDays`. Перемещение в карантин само по себе не освобождает место.
- Пути источника и назначения должны существовать, не быть symlink, не совпадать
  и не находиться один внутри другого.

## Расписание

Все задания выполняются последовательно в часовом поясе `APP_TIMEZONE`.

| Время | Операция |
|---|---|
| Каждый час | Синхронизация текущего дня |
| 22:00 | Финальная синхронизация вчерашнего дня |
| 03:00 | Контроль лимита источника |
| 04:00 | Контроль лимита хранилища |
| 05:00 | Удаление старых записей журналов |
| 06:00 | Карантин и очистка карантина |

## API

| Endpoint | Метод | Назначение |
|---|---|---|
| `/api/config` | GET | Текущая конфигурация |
| `/api/config` | POST | Частичное обновление конфигурации |
| `/api/space` | GET | Использование обоих дисков |
| `/api/copy` | GET | Журнал копирования |
| `/api/delete` | GET | Журнал обслуживания и карантина |
| `/api/errors` | GET | Журнал ошибок |
| `/api/health` | GET | Состояние сервера, БД и каталогов |

API рассчитан на доверенную внутреннюю сеть. Для публикации наружу используйте
reverse proxy с аутентификацией и TLS.

## Docker

Измените реальные пути `/mnt/ftp` и `/mnt/smb` в `docker-compose.yml`, затем:

```bash
docker compose up -d --build
docker compose logs -f
```

Конфигурация, SQLite и WAL-файлы сохраняются в `./data`.

При обновлении со старой версии:

```bash
mkdir -p data
mv config.json autoexport.db autoexport.db-shm autoexport.db-wal data/ 2>/dev/null || true
docker compose up -d --build
```

## Массовое копирование

```bash
bun run copy-all
```

Не запускайте `copy-all` одновременно с плановыми файловыми операциями сервера.

## Подключение сетевых ресурсов

`mount-network.sh` умеет настраивать NFS, SMB и SSHFS. Для SMB пароль не хранится
в репозитории:

```bash
sudo SMB_PASSWORD='секрет' ./mount-network.sh
```
